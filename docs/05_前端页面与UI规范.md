# 05. 前端页面与 UI 规范

## 1. 整体风格

系统面向广告公司内部使用，建议采用：

- 红黑科技风
- 深色后台
- 高对比信息卡片
- 清晰的状态颜色
- 卡片式经营看板
- 表格密度适中
- 重点金额醒目展示

## 2. 推荐 UI 技术

- Vue 3
- Vite
- TypeScript
- Element Plus
- Pinia
- Vue Router
- Axios
- ECharts

## 3. 布局

```text
顶部：系统名称、搜索、通知、用户菜单
左侧：主导航菜单
中间：业务内容区
右侧：可选提醒栏
```

## 4. 主菜单

```text
首页驾驶舱
客户管理
报价管理
订单管理
设计管理
生产管理
安装管理
外协管理
采购库存
财务管理

```

### 顶部数据卡片

- 本月未收款

字段列：

- 客户名称
- 联系人
- 电话
- 类型
- 等级
- 总订单金额
- 未收金额
- 最近下单日期

- 查看详情

- 新建订单
- 生成对账单

## 7. 报价单页面

报价单页面是重点，必须易用。

### 页面结构

```text
客户信息区
项目信息区
报价明细表格
费用汇总区
备注条款区
操作按钮区
```

### 报价明细表格

字段：

- 产品
- 材质
- 工艺
- 长
- 宽
- 数量
- 单位
- 面积
- 单价
- 小计
- 备注

### 按钮

- 添加明细
- 复制明细
- 删除明细
- 重新计算
- 保存草稿
- 导出 PDF
- 客户确认
- 转订单

## 8. 订单详情页

订单详情页建议采用标签页。

标签页：

- 基本信息
- 报价明细
- 设计任务
- 制作任务
- 安装任务
- 外协任务
- 收款记录
- 文件资料
- 操作日志

顶部显示订单状态流程条。

## 9. 设计任务页面

列表视图：

- 任务编号
- 订单编号
- 客户名称
- 设计内容
- 设计师
- 截止时间
- 状态
- 操作

详情页：

- 需求说明
- 客户素材
- 参考图
- 设计稿上传
- 修改意见时间线

## 10. 制作任务页面

建议支持列表 + 看板。

看板列：

```text
待制作 | 排队中 | 制作中 | 待质检 | 返工 | 已完成
```

任务卡片显示：

- 客户
- 产品
- 尺寸
- 数量
- 截止时间
- 负责人

## 11. 安装任务页面

PC 端：安装派工与日程管理。

手机端：安装人员操作。

手机端重点按钮：

- 查看地址
- 拨打联系人电话
- 上传安装前照片
- 上传安装中照片
- 上传安装后照片
- 客户签字
- 提交验收

## 12. 财务页面

页面：

- 收款登记
- 欠款列表
- 客户对账
- 支出记录
- 发票记录

欠款列表需要突出：

- 客户名称
- 未收金额
- 最早欠款日期
- 超期天数
- 业务员

## 13. 状态颜色规范

```text
红色：逾期、欠款、返工、异常
黄色：待处理、待确认、待审核
蓝色：进行中
绿色：完成、已收款、已验收
灰色：取消、作废、归档
```

## 14. 响应式要求

第一版 PC 优先。  
安装任务和照片上传必须兼容手机浏览器。

## 15. UI 注意事项

- 金额字段右对齐。
- 日期字段统一格式：YYYY-MM-DD HH:mm。
- 状态字段使用 Tag 标签。
- 重要操作需要确认弹窗。
- 删除优先软删除。
- 导出按钮集中放在右上角。
- 表格需要搜索、筛选、分页。
