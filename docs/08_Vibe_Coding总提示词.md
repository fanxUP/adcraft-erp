# 08. Vibe Coding 总提示词

下面这段可以直接复制给 Vibe Coding / Codex 作为项目总提示词。

---

你是一名资深全栈工程师和产品架构师。请根据以下要求，开发一套广告制作安装工程管理系统。

## 项目定位

开发一套局域网部署的广告行业业务管理系统，客户机通过浏览器访问。系统用于广告公司管理客户、报价、订单、设计、制作、安装、外协、收款、对账、文件归档和经营报表。

系统不是简单开单软件，而是适合广告制作安装工程业务的轻量级 ERP + 生产任务管理 + 安装闭环系统。

## 技术栈

- 前端：Vue 3 + Vite + TypeScript + Element Plus
- 状态管理：Pinia
- 路由：Vue Router
- 图表：ECharts
- 后端：FastAPI + Python
- ORM：SQLAlchemy 2.x
- 数据库迁移：Alembic
- 数据库：PostgreSQL
- 缓存：Redis
- 文件存储：MinIO 或本地 uploads 目录，优先预留 MinIO
- 网关：Nginx
- 部署：Docker Compose

## 部署要求

系统部署在公司局域网服务器，不依赖公网云服务。员工电脑通过浏览器访问，例如：

```text
http://192.168.1.10
http://erp.local
```

8. 报价转订单。

10. 订单状态流转。
11. 设计任务管理。

## 核心业务流程

```text
客户建档 → 新建报价 → 报价确认 → 报价转订单 → 生成设计/制作/安装任务 → 上传设计稿/生产文件/安装照片 → 安装验收 → 登记收款 → 客户对账 → 报表统计
```

## 报价要求

报价模块必须适合广告行业，支持以下计算方式：

1. 按面积：长 × 宽 × 数量 × 单价。
2. 按数量：数量 × 单价。
3. 按长度：长度 × 数量 × 单价。
4. 按字数：字数 × 单价。
5. 工程分项：材料费 + 加工费 + 安装费 + 运输费 + 管理费 + 利润。

报价公式不要写死，要尽量通过产品、材质、工艺、报价规则配置实现。

## 订单要求

报价单确认后可以转正式订单。订单必须有状态流转和状态日志。订单详情页需要展示：基本信息、报价明细、设计任务、制作任务、安装任务、外协任务、收款记录、文件资料、操作日志。

## 安装要求

安装任务必须支持手机网页操作。安装人员可以查看安装地址、联系人、安装要求，可以上传安装前照片、安装中照片、安装后照片，可以提交验收记录。第一版客户签字可以用上传图片或简单签名字段实现。

## 财务要求

支持多笔收款、部分收款、订单未收金额自动计算、客户欠款统计、客户对账单。收款记录不允许物理删除，只能作废，并保留操作日志。

## 文件要求

所有文件必须绑定业务对象，例如客户、报价、订单、设计任务、制作任务、安装任务、收款记录。订单资料需要统一归档。

## UI 风格

界面采用红黑科技风，适合广告公司使用。首页需要经营驾驶舱，订单状态要清晰，表格要支持搜索、筛选、分页。PC 端优先，安装任务手机端必须可用。

## 开发要求

- 代码结构清晰，模块化。
- 后端分层：router、schema、model、service、repository。
- 前端分层：api、views、components、stores、router。
- 所有接口要有基础错误处理。
- 所有金额使用 decimal/numeric，不要使用 float。
- 所有核心表有 created_at、updated_at。
- 关键业务操作写 operation_logs。
- 提供 Docker Compose 一键启动。
- 提供 README，写清楚本地开发和局域网部署方法。

## 开发顺序

请按阶段开发，不要一次性堆所有功能：

1. 项目初始化、Docker、登录、权限。
2. 客户、产品、材质、工艺、报价。
3. 报价转订单、订单状态、任务管理。
4. 文件上传、设计稿、安装照片、验收。
5. 收款、欠款、客户对账。
6. 首页驾驶舱、基础报表。
7. 外协、库存、利润。
8. 权限细化、备份、优化和上线。

每完成一个阶段，先保证可运行、可测试、可回滚，再进入下一阶段。
