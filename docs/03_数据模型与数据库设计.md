# 03. 数据模型与数据库设计

## 1. 设计原则

- 所有核心业务对象必须有唯一编号。
- 报价转订单后，报价版本快照必须保留。
- 订单状态变更必须记录日志。
- 收款记录不允许物理删除，只能作废。
- 文件必须绑定业务对象。
- 所有表保留 created_at、updated_at、deleted_at。

## 12. 状态枚举建议

### 报价状态

```text
draft, pending_review, approved, sent, confirmed, converted, cancelled
```

### 订单状态

```text
pending_confirm, confirmed, pending_design, designing, pending_production, producing, pending_outsourcing, outsourcing, pending_installation, installing, pending_acceptance, completed, pending_payment, partially_paid, paid, cancelled
```

### 设计任务状态

```text
pending_assign, pending_design, designing, pending_customer_confirm, revision_required, confirmed, archived
```

### 制作任务状态

```text
pending, queued, producing, pending_qc, qc_passed, reworking, completed
```

### 安装任务状态

```text
pending_dispatch, dispatched, on_the_way, installing, pending_acceptance, accepted, rework_required, completed
```

## 13. 金额字段约定

所有金额字段使用 `numeric(14,2)`。

不要使用 float/double 存金额。

## 14. 尺寸字段约定

长度、宽度、高度建议使用毫米或米。第一版建议统一使用米，显示时保留两位或三位小数。

## 15. 软删除

重要业务表使用 `deleted_at` 字段软删除。

财务表和状态日志不建议删除。
