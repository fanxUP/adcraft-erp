# 04. 重构与代码规范提示词

请对当前系统进行代码质量检查和重构，但不要破坏已完成业务功能。

## 后端规范

目录建议：

```text
backend/app/
  api/
  core/
  models/
  schemas/
  services/
  repositories/
  utils/
  tests/
```

要求：

- router 只处理请求和响应，不写复杂业务逻辑。
- service 负责业务逻辑。
- repository 负责数据库访问。
- schema 负责请求和响应模型。
- model 负责 SQLAlchemy 模型。
- 金额字段使用 Decimal。
- 关键操作写 operation_logs。
- 异常统一处理。

## 前端规范

目录建议：

```text
frontend/src/
  api/
  assets/
  components/
  layouts/
  router/
  stores/
  utils/
  views/
```

要求：

- API 请求统一封装。
- 表格组件可复用。
- 状态 Tag 可复用。
- 金额格式化可复用。
- 日期格式化可复用。
- 页面不要堆超长逻辑。

## 安全规范

- 密码必须 hash。
- JWT secret 从环境变量读取。
- 文件上传限制大小和类型。
- 财务操作必须鉴权。
- 恢复备份必须管理员二次确认。

## 重构验收

- 原有功能测试通过。
- Docker Compose 正常启动。
- 登录、报价、订单、收款主流程不受影响。
- 代码目录更清晰。
- README 更新。
